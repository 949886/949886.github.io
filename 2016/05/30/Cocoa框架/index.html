字符比较
  • -(NSComparisonResult)compare:(NSString *)string;
  • -(NSComparisonResult)localizedCompare:(NSString *)string;

localizedCompare: 方法会根据当前系统语言选择进行比较的方法，如简体中文环境按拼音排序，繁体中文环境按笔画排序。返回结果：

  • NSOrderedAscending 当前字符靠前
  • NSOrderedSame 相同
  • NSOrderedDescending 当前字符靠后

用localizedCompare:方法对字符串数组进行排序：

NSArray *array = @[@"超人", @"Superman", @"蝙蝠侠", @"Batman", @"X战警", @"X-man", @"钢铁侠", @"Ironman"];
NSArray *sortedArray = [array sortedArrayUsingSelector:@selector(localizedCompare:)];
for (NSString * string in sortedArray) NSLog(@"%@", string); 
//Output(zh_CN): 蝙蝠侠 超人 钢铁侠 Batman Ironman Superman X-man X战警

NSScanner

获取两个字符串间的字符串 
@interface NSString (Extension)
- (NSString *)substringWithStart:(NSString *)start End:(NSString *)end
{
    if(start && ![self containsString:start]) return nil;
    if(end && ![self containsString:end]) return nil;

    NSString * result;
    NSScanner * scanner = [NSScanner scannerWithString:self];
    if(nil != start) {
        [scanner scanUpToString:start intoString:&result];
        [scanner scanString:start intoString:&result];
    }
    if(nil == end) end = @"\0";
    [scanner scanUpToString:end intoString:&result];

    return result;
}
@end  

NSString * string = @"https://api.example.com/parms?text=a&query=search";
NSString * substring = [string substringWithStart:@"query=" End:nil];
NSLog(@"%@", substring); //Output:search

scanString:intoString: 从当前扫描位置开始出现相同字符串则返回YES并通过过第二个参数返回该参数，否则返回NO。
scanUpToString:intoString: 方法会对字符串进行扫描直到遇到指定字符串出现，若出现返回YES且将扫描经过的字符串通过第二个参数返回，扫描位置会移至扫描到字符串前，否则返回NO。

NSRegularExpression

NSRegularExpression是Cocoa中用于处理正则表达式的API（什么是正则表达式，请参考这篇文章：正则表达式30分钟入门教程）。
下面通过一个例子来可以轻松理解NSRegularExpression的使用。该程序的功能为匹配一对中括号（中括号中可以有任意长度的内容，但是不包括成对的中括号），并将匹配到的内容全部替换成「<表情>」。

NSString * pattern = @"\\[[^ \\[\\]]+?\\]";
NSString * string = @"[哆啦A梦害怕]好可怕[哆啦A梦害怕]我得走";
NSString * replacement = @"<表情>"; //将匹配到的字符串替换成该字符串 

NSRegularExpression * regex = [NSRegularExpression regularExpressionWithPattern:pattern options:NSRegularExpressionCaseInsensitive error:nil];
NSArray * matchs = [regex matchesInString:string options:NSMatchingReportCompletion range:NSMakeRange(0, string.length)]; 

int offset = 0; //替换完后源字符串将变短，用该值进行补偿
for (NSTextCheckingResult * match in matchs)  {
        NSRange range = NSMakeRange(match.range.location - offset, match.range.length);
        string = [string stringByReplacingCharactersInRange:range withString: replacement];
        offset += range.length - replacement.length;
} 

NSLog(@"Result string: %@", string); //Output: <表情>好可怕<表情>我得走

NSCalendar

NSTimer

初始化：

+(NSTimer *)timerWithTimeInterval:(NSTimeInterval)ti target:(id)aTarget selector:(SEL)aSelector userInfo:(nullable id)userInfo repeats:(BOOL)yesOrNo;
+(NSTimer *)scheduledTimerWithTimeInterval:(NSTimeInterval)ti target:(id)aTarget selector:(SEL)aSelector userInfo:(nullable id)userInfo repeats:(BOOL)yesOrNo;

timerWithTimeInterval: 创建NSTimer的基本方法（需要加入RunLoop才会生效）
scheduledTimerWithTimeInterval: 会创建一个NSTimer，并将timer自动加入RunLoop中，且Mode为NSDefaultRunLoopMode

将Timer加入RunLoop 

NSTimer * timer = [NSTimer timerWithTimeInterval:interval target:target selector:selector userInfo:nil repeats:true];
NSRunLoop * currentLoop = [NSRunLoop currentRunLoop];
[currentLoop addTimer:timer forMode:NSDefaultRunLoopMode];﻿​

Mode: 选择在RunLoop的那种模式下运行，常用值如下

  • NSDefaultRunLoopMode 默认情况（滚动时timer将不运行）
  • UITrackingRunLoopMode 当UIScrollView发生滚动的情况
  • NSRunLoopCommonModes 主线程RunLoop的CommonModes包括以上两种Mode，即滚动和默认情况都会运行。
Timer的释放

将Timer 添加到 Runloop ，Runloop会对Timer强引用，而Timer又拥有target(一般为self)的强引用，会造成self无法释放。故在self释放前需要对timer进行invalidate。

Timer在Controller中：

- (void) viewWillDisappear:(BOOL)animated {
    [self viewWillDisappear:animated];
    [self.timer invalidate];
    self.timer = nil;
}

Timer在View中：

-(void)removeFromSuperview {
    [super removeFromSuperview];
    [self.timer invalidate];
    self.timer = nil;
}
暂停与恢复Timer

暂停： timer.fireDate = [NSDate distantFuture];
恢复： timer.fireDate = [NSDate date];


文件

NSFileManager

NSFileManager 是 Foundation 框架中处理文件系统API。采用单例设计模式：
+(NSFileManager *)defaultManager;﻿​

文件是否存在
  • -(BOOL)fileExistsAtPath:(NSString *)path;
  • -(BOOL)fileExistsAtPath:(NSString )path isDirectory:(nullable BOOL )isDirectory;
  • -(nullable NSArray )subpathsAtPath:(NSString )path;
计算文件及文件夹大小 

@interface NSFileManager (Extension)
+(long long)folderSizeWithPath:(NSString *)path {
    long long size = 0;
    NSFileManager * manager = [NSFileManager defaultManager];
    if(![manager fileExistsAtPath:path]) return 0; 

    NSArray<NSString *> * subpaths = [manager subpathsAtPath:path];
    for (NSString * subpath in subpaths) {
        NSString * fullpath = [path stringByAppendingPathComponent:subpath];
        size += [self fileSizeWithPath:fullpath];
    }
    return size;
} 

+(long long)fileSizeWithPath:(NSString *)path {
    NSFileManager *manager = [NSFileManager defaultManager];
    if (![manager fileExistsAtPath:path]) return 0;
    return [[manager attributesOfItemAtPath:path error:nil] fileSize];
}
@end

注意：计算出来文件大小的单位为字节，系统计算大小的进制为1000，故要以MB显示需要将结果除以1,000,000。

获取文件属性 

NSFileManager * manager = [NSFileManager defaultManager];
NSDictionary<NSString *, id> * attributes = [manager attributesOfItemAtPath:path error:nil];
long long fileSize = attribute[NSFileSize]; //通过Key获得属性值（以FileSize为例）
long long fileSize = [attribute fileSize];  //通过NSDictionary的NSFileAttributes Category属性获得属性值（结果相同）

常用文件属性：

  • NSFileSize 文件字节大小
  • NSFileType 文件类型 ，常用值如下
    • NSFileTypeRegular 普通文件
    • NSFileTypeDirectory 文件夹
    • NSFileTypeSymbolicLink 快捷方式
    • NSFileTypeUnknown 未知
  • NSFileCreationDate 创建日期
  • NSFileModificationDate 修改日期
  • NSFileSystemSize 系统磁盘大小（Category无对应方法）
  • NSFileSystemFreeSize 系统磁盘剩余大小（Category无对应方法）

NSXMLParser

XML文件的解析有两种方式：SAX (Simple API for XML)、DOM (Document Object for Model)。

  • SAX解析器是事件驱动的，他会顺序解析XML文件中的元素。适合解析大文件。
  • DOM解析器会将整个XML文档以树的形式读取到内存中，会消耗较多的内存资源，XML文件越大弊端越明显。故适合解析小文件。

Cocoa框架自然也都提供了这两种方式的解析器，而作为DOM解析器的XML处理类NSXMLDocument、NSXMLElement、NSXMLNode只能在OS X系统中使用，在iOS中可以使用支持SAX方式解析的NSXMLParser类，或者基于纯C语言的libxml库（支持SAX与DOM解析）。

通过NSXMLParser解析XML，并打印xml内容：

@interface XMLParser : NSObject <NSXMLParserDelegate>
+ (void)parse:(NSString *)xml;
@end 

@implementation XMLParser 
+ (void)parse:(NSString *)xml
{
    NSData * data = [xml dataUsingEncoding:NSUTF8StringEncoding];
    NSXMLParser * parser = [[NSXMLParser alloc]initWithData:data];
    XMLParser * delegate = [XMLParser new];
    parser.delegate = delegate;
    [parser parse];
} 
//开始解析xml文件时调用
- (void)parserDidStartDocument:(NSXMLParser *)parser {
    NSLog(@"parserDidStartDocument");
} 
//结束解析xml文件时调用
- (void)parserDidEndDocument:(NSXMLParser *)parser {
    NSLog(@"parserDidEndDocument");
} 
//解析到一个元素的开始时调用，如<a href="http://www.example.com">。
- (void)parser:(NSXMLParser *)parser didStartElement:(NSString *)elementName namespaceURI:(nullable NSString *)namespaceURI qualifiedName:(nullable NSString *)qName attributes:(NSDictionary<NSString *, NSString *> *)attributeDict {
    __block NSString * attributes = @"";
    [attributeDict enumerateKeysAndObjectsUsingBlock:^(NSString * _Nonnull key, NSString * _Nonnull obj, BOOL * _Nonnull stop) {
        attributes = [NSString stringWithFormat:@"%@ %@=\"%@\"", attributes, key, obj];
    }];
    printf("<%s %s>", elementName.UTF8String, attributes.UTF8String);
} 
//解析到一个元素的结束时调用，如</a>
- (void)parser:(NSXMLParser *)parser didEndElement:(NSString *)elementName namespaceURI:(nullable NSString *)namespaceURI qualifiedName:(nullable NSString *)qName {
    printf("</%s>", elementName.UTF8String);
} 
//解析到元素的内容如<a href="http://www.example.com">Link</a>中的Link时调用
- (void)parser:(NSXMLParser *)parser foundCharacters:(NSString *)string {
    printf("%s", string.UTF8String);
}
@end
