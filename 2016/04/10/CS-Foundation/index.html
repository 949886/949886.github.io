1. const的含义

const int a;
int const a;
int const a;
int const const a;

1> 第一、二个意味着 a 是一个指向常整型数的指针(整型数是不可修改的，但指针可以)
2> 第三个的意思：a 是一个指向整型数的常指针(指针指向的整型数是可以修改的，但指针是不可修改的)
3> 最后一个意味着：a 是一个指向常整型数的常指针(指针指向的整型数是不可修改的，同时指针也是不可修改的)

快排的基本思想（摘自维基百科）：
使用分治法（Divide and conquer）策略来把一个序列（list）分为两个子序列（sub-lists）。步骤为

  1. 从数列中挑出一个元素，称为「基准」（pivot），
  2. 重新排序数列，所有元素比基准值小的摆放在基准前面，所有元素比基准值大的摆在基准的后面（相同的数可以到任一边）。在这个分区结束之后，该基准就处于数列的中间位置。这个称为分区（partition）操作。
  3. 递归地（recursive）把小于基准值元素的子数列和大于基准值元素的子数列排序。

2. 说说你对事务的理解

在需要对数据库进行批量操作时，使用事务可以保证数据的统一和完整性，同时也可以提高效率。即使在执行的过程中出现了错误也可以通过对事务进行回滚而放弃这次操作。

事务具有以下特性，这些性质通常被称为 ACID 特性 (ACID property)：

  • Atomic（原子性）：事务中的所有操作操作要么全部成功，要么全部失败。
  • Consistency（一致性）：如果数据库开始是一致的，则事务执行之后数据库仍处于一致状态。一致状态是指数据库中的数据应满足某个特定的完整性约束。保证数据库一致性是程序员的责任。
  • Isolation（隔离性）：虽然多个事务能够并发执行，但系统会保证：对于任意事务T1和T2，T2会在T1开始之前执行完毕，或者在T1完成之后开始执行。
  • Durability（持久性）：事务成功完成后，它对数据库的改变是永久的。

操作系统

1. 堆和栈的区别

栈内存是由系统自动分配和释放的，而堆内存需要程序员手动管理。以C++为例，用new关键字创建一个对象则是在堆上申请了一块内存，该内存必须由程序有手动调用delete释放，否则会一直存在于内存中。而创建一个基础类型则是在栈上分配了一块内存，在域结束时该栈上的内存会自动释放。
在Win下，栈是向低地址扩展的数据结构，是一块连续的内存区域；堆是向高地址扩展的数据结构，是不连续的内存区域。

2. 进程和线程的区别

进程是操作系统给程序进行资源分配的基本单位；线程是程序进行CPU调度和分配的基本单位。可以这么说：进程是程序的一次执行，线程是程序的一段片段。
一个程序可以有一个或多个进程；一个进程至少有一个线程。
进程有独立的地址空间；而同一个进程中的线程的内存空间是共享的，它们共享该进程中的内存空间，并发效率较高。

TCP / UDP 的区别以及使用场景

TCP 和 UDP都是传输层协议。

TCP协议适用于对效率要求相对低，但对准确性要求相对高的场景下，或者是需要连接的场景下；而UDP协议适用于对效率要求相对高，对准确性要求相对低的场景。
Udp相比tcp也就是少了启动时三次握手，以及不进行数据包传输校验。
