线程

线程的状态

一个有生命的线程具有「创建」「就绪」「运行」「阻塞」「死亡」共五种状态。

NSThread

NSThread为Objective-C提供了显示的创建和管理线程的API，其是对底层POSIX 线程C语言API的面向对象封装。

线程操作


获取线程

+(NSThread *)mainThread; //获取主线程
+(NSThread *)currentThread; //获取当前线程

创建新线程

-(id)initWithTarget:(id)target selector:(SEL)selector object:(id)argument;
+(void)detachNewThreadSelector:(SEL)aSelector toTarget:(id)aTarget withObject:(id)anArgument;

开启&关闭线程

-(void)start;
-(void)cancel;

挂起线程

+(void)sleepUntilDate:(NSDate *)date;
+(void)sleepForTimeInterval:(NSTimeInterval)ti;

挂起线程之后线程将进入阻塞状态

停止线程

+(void)exit;
调用[NSThread exit]可以强制停止当前线程。

NSObject多线程操作

@param aSelector:要执行的方法 arg:SEL的第一个参数或nil modes:RunLoop Modes

@synchronized

The @synchronized directive is a convenient way to create mutex locks on the fly in Objective-C code. The @synchronized directive does what any other mutex lock would do—it prevents different threads from acquiring the same lock at the same time. In this case, however, you do not have to create the mutex or lock object directly. Instead, you simply use any Objective-C object as a lock token, as shown in the following example:

The object passed to the @synchronized directive is a unique identifier used to distinguish the protected block. If you execute the preceding method in two different threads, passing a different object for the anObj parameter on each thread, each would take its lock and continue processing without being blocked by the other. If you pass the same object in both cases, however, one of the threads would acquire the lock first and the other would block until the first thread completed the critical section.
As a precautionary measure, the @synchronized block implicitly adds an exception handler to the protected code. This handler automatically releases the mutex in the event that an exception is thrown. This means that in order to use the @synchronized directive, you must also enable Objective-C exception handling in your code. If you do not want the additional overhead caused by the implicit exception handler, you should consider using the lock classes.
For more information about the @synchronized directive, see a target=”_self” The Objective-C Programming Language/a.

NSOperation

Cocoa框架的Operation是一种面向对象方法，用于封装需要处理的异步任务，其是对GCD底层C语言API的面向对象封装。
NSOperation是一个抽象类，无法直接使用。它给了它的子类一个十分有用而且线程安全的方式来建立状态、优先级、依赖性和取消等的模型。Cocoa框架提供了两个继承自NSOperation的子类，NSInvocationOperation 与 NSBlockOperation 以进行异步任务处理。也可以使用自定义的NSOperation的子类以满足更多需求。

NSOperation有三种状态：

  • isReady：准备执行
  • isExecution：正在执行
  • isFinished：执行完毕或被取消

-(void)start;
-(void)cancel;

@property (nullable, copy) void (^completionBlock)(void);

NSBlockOperation

一个可以并发执行多个block对象的类。当所有block全都执行完毕后该Operation才被视为执行完毕。

  • +(instancetype)blockOperationWithBlock:(void (^)(void))block;
  • -(void)addExecutionBlock:(void (^)(void))block;

NSInvocationOperation

可以通过一个对象及Selector创建一个这样的operation对象。当有一个需要被执行的方法，你可以使用该类调用并执行任务。

  • -(instancetype)initWithInvocation:(NSInvocation *)inv;
  • -(nullable instancetype)initWithTarget:(id)target selector:(SEL)sel object:(nullable id)arg;

自定义Operation

可以通过重写 main 或者 start 方法 来定义自己的 operations 。

Grand Central Dispatch (GCD)

对于异步任务执行，Objective-C中一般有两个选择：Grand Central Dispatch或者NSOperation。而GCD已经逐渐成为主流，用于处理各种异步操作。
