1. @property中assign,copy,retain,strong,weak用法与区别

各个属性应该用在哪？
assign: 基本类型
copy: NSString, NSArray, Block 等
strong: Common object
weak: UIKit / Delegate

strong 与 weak 的区别
strong：为强引用。之所以称之为“强”引用，是因为它会保持住实例，只要强引用还在，实例是不允许被销毁的。
weak：为弱引用。如果一个实例只有弱引用，则其将被释放。为此种属性设置新值时，既不retain也不release，当对象被销毁时属性值被清空（nil out）。

assign, copy, retain的区别
assign: 对基本值类型（scalar type，如NSInteger与C基本类型）进行简单的赋值操作。
copy: 深拷贝新对象，然后释放旧对象。
retain: 浅拷贝赋值对象的地址，其引用计数+1，旧对象引用计数-1。

assign 与 weak 的区别
weak用于对象且在对象被销毁时会被属性值被清空（nil out）。

2. 谈谈消息转发机制（动态解析）

Objective-C在发送消息时会Runtime会根据实例的isa指针所指向的Class及该Class的父类中找到对应的方法列表，若在最顶层的父类中仍然未找到SEL对应的方法则会触发消息转发机制：

  1. Objective-C会调用该类的 +(BOOL)resolveInstanceMethod:(SEL)sel 和 +(BOOL)resolveClassMethod:(SEL)sel 来指定是否动态添加方法（返回YES则需要在方法内手动通过class_addMethod添加对应方法，返回NO则会进入下一步）。
  2. 若第一步返回NO，则Runtime会检查类是否实现了 -/+(id)forwardingTargetForSelector:(SEL)aSelector 方法(该方法需要返回一个Class，实现-处理转发实例方法，实现+则转发类方法)，用于将该消息转发给其他类处理。
  3. 若第二步返回nil，则Runtime会检查类是否实现了 -(NSMethodSignature *)methodSignatureForSelector:(SEL)aSelector 和 -(void)forwardInvocation:(NSInvocation *)anInvocation方法（若处理的是类方法则为+）。Runtime会先通过methodSignatureForSelector: 来获取方法签名，若此方法返回nil则程序就会抛出unrecognized selector异常，若返回了一个方法签名则会调用forwardInvocation:来手动处理该方法签名对应的invocation。

3.什么是Method Swizzling？

Method Swizzling 可以交换两个方法的实现。方法由两个部分组成，分别是SEL和IMP。SEL为方法的名称，IMP是方法的实现。通过Method Swizzling可以不通过继承而重写原方法，并且还可以保留原方法的实现。

参考：NSHisper - Method Swizzling

4. +load和+initialize方法的用处及区别

+load和+initialize都是用于类初始化的方法。

+(void)load

类载入时会调用。在执行子类的load方法之前，必定会先执行超类的load方法。因为无法判断各个类的载入顺序，故在load方法中使用其他类是不安全的。
如果某个类没有实现load方法，那么无论其超类有没有实现load方法，该类都不会超类的load方法。分类中也可以定义load方法，类的load方法要比其分类中的先调用。
通常Method-Swizzling会在+load方法中进行。

+(void)initialize

类首次被使用之前调用。该方法时「惰性调用」的，也就是说只有当程序用到相关类时才会调用，如果一个类一直都没被使用，那么该方法就一直不会被调用。
如果某个类未实现该方法，而其超类实现了该方法，则该类就会调用超类的initialize方法。鉴于此，一般都会这样实现实现initialize方法：
+(void) initialize { if (self == [ClassName class]) { /* initialize */ } }
